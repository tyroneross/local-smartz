Local Smartz CLI Flows

Calm Precision applied to terminal UX — hierarchy, disclosure, signal, error strategy

Flow 1 — Install

Clone and run one script. Progressive output: each step confirms before moving to next.

Terminal — install.sh
$ git clone https://github.com/tyroneross/local-smartz.git $ cd local-smartz && bash install.sh Local Smartz Installer ══════════════════════ Python 3.12: OK Ollama: installed Detected: running inside cloned repo Installing in editable mode... Installed: localsmartz 0.1.0 Setting up Ollama and downloading models... Ollama: running (v0.15.2) Profile: full qwen3:8b-q4_K_M: ready gpt-oss:120b: ready Done! Run localsmartz to pick your model and start researching.
Design Three-line hierarchy: bold title, status lines (color-only, no badges), final CTA. Each step confirms with green before advancing. Errors halt immediately with what-why-fix.
Hierarchy Signal > Noise Progressive Disclosure Natural Language

Flow 2 — First Run (Model Picker)

No config exists. Before running the query, guide user to pick their model. One decision, then go.

Terminal — first run
$ localsmartz "What is quantum computing?" Local Smartz v0.1.0 First-run setup Ollama: running (v0.15.2) Hardware: 128 GB RAM · full profile Available models: 1. qwen3:8b-q4_K_M 5.2 GB 2. gpt-oss:20b 13.8 GB 3. gpt-oss:120b 65.3 GB <- recommended Select model [3]:
Design Minimal chrome. Model list is a clean table — no borders on individual items, just whitespace alignment. Recommendation is inline text color (accent), not a badge. Default in brackets = one-key confirmation. Sizes are muted metadata.
Hick's Law Grouping Text > Decoration Size = Importance

Flow 2b — After Selection

User presses Enter (accepts default). Config saved, query executes immediately.

Terminal — running research
Select model [3]: 3 Saved -> .localsmartz/config.json Profile: full | Model: gpt-oss:120b Query: What is quantum computing? --- ▸ write_todos(plan research steps) ▸ web_search(query=quantum computing overview 2026) ▸ scrape_url(url=https://en.wikipedia.org/wiki/Q...) ▸ create_report(title=Quantum Computing Overview) --- Tools used: create_report, scrape_url, web_search, write_todos Quantum computing uses quantum mechanical phenomena — superposition and entanglement — to process information in ways that classical computers cannot efficiently replicate... [Report saved to .localsmartz/reports/quantum-computing.md]
Design Streaming progress to stderr (tool calls with purple markers). Clean separation: metadata above the line, tool progress between lines, final answer below. Tool previews show just enough args to be useful without clutter.

Flow 5 — Error States

What-why-fix pattern. Each error is self-contained with a recovery action.

Ollama not running
$ localsmartz "What is AI?" Ollama is not running. Start it with: ollama serve Or open the Ollama app.
No models available
$ localsmartz "What is AI?" Ollama: running (v0.15.2) No models found in Ollama. Pull a model first: ollama pull qwen3:8b-q4_K_M (5 GB, fast) ollama pull llama3.1:70b (40 GB, powerful)
Saved model removed
$ localsmartz "What is AI?" Saved model 'gpt-oss:120b' is no longer available. Available models: 1. qwen3:8b-q4_K_M 5.2 GB 2. gpt-oss:20b 13.8 GB <- recommended Select model [2]:
Tool error during research
▸ web_search(query=quantum computing) ✗ Error: DuckDuckGo rate limited (429) ▸ scrape_url(url=https://en.wikipedia.org/...) --- Tools used: scrape_url, web_search, write_todos Drift events: 1 ⚡ error_burst [low] web_search: 1 error in 1 calls
Error Pattern Every error follows what-why-fix. Red = stop (blocking error). Yellow = warning (recoverable). Green preceding = what worked. No stack traces — user-facing copy only. Recovery actions are copy-pasteable commands.
Error: What-Why-Fix Signal > Noise Natural Language Functional Integrity

Flow 6 — Lite Profile Guardrails

Smaller models get tighter constraints. Drift detection and loop breaking surface visually.

Terminal — lite profile with loop detection
$ localsmartz --profile lite "Analyze Tesla stock performance" Profile: lite | Model: qwen3:8b-q4_K_M Query: Analyze Tesla stock performance --- ▸ write_todos(plan stock analysis) ▸ web_search(query=Tesla stock price 2026) ▸ scrape_url(url=https://finance.yahoo.com/...) ▸ web_search(query=Tesla stock price 2026) ⚠ Loop detected: web_search called 3x with same args — breaking --- Tools used: scrape_url, web_search, write_todos Drift events: 1 ⚡ stale_search [medium] web_search: repeated identical query Based on available data, Tesla's stock has shown...
Design Guardrails are visible but not alarming. Yellow warning icon for loop break — the system handled it, not a crash. Drift events summarized at end for transparency. User sees the research still completed despite the loop.
